hugo 需知

建立站点

hugo new site

文件结构

  • config.toml: 站点根目录配置文件1
  • archetypes/: 新文章的模板文件2
  • content/: 网站的所有内容
  • data/: 作为内容补充的数据文件3
  • layouts/: html 模板
  • static/: 图片,css ,js 等静态文件，该目录下的文件会直接拷贝到/public
  • themes/: 所有主题
  • public/: 执行 hugo 命令后，生成的静态文件

_index.md

https://gohugo.io/content-management/organization/#path-breakdown-in-hugo

简单来说 “_index.md” 是作为该目录下的所有文章的 Front Matter 共享模板5

类型模板

hugo 的 type 属性默认为 content，即实际文章所在的父目录名

如果我们在 Front Matter 中添加以下代码

1
2

type: posts
layout: newspaper

那么此文章将会最优先寻找 /layouts/posts/newspaper.html 作为模板应用
